Developing Your Pitch

1. Identify the domain (subject area)

Use an appropriate amount of specificity.

Tip‚úÖ Good examples of domains
  • Explainable AI
  • Healthcare economics
  • Social network analysis
Warning‚ùåBad examples of domains
  • Machine Learning (too broad)
  • Minor 7th dominant cords in Bach symphonies (too specific)
  • Data visualization (too meta)

2. Describe a target user of your vis

Use objective observables to describe them that relate to the task (check Chapter 3).

Tip‚úÖ Good examples of users
  • Consumer with a fitness tracker
  • A potential voter
  • Venture capitalist
Warning‚ùå Bad examples of users
  • Myself and/or my friends
  • Someone that doesn‚Äôt understand the data
  • The professor grading this vis

3. Describe the main task(s)

These could be action/target pairs or analysis question(s) the user of your vis would need to do (check Chapter 3).

Tip‚úÖ Good examples of tasks
  • Determine how object size affects semantic segmentation boundaries
  • Cluster Goodreads users based on book genre and recommendations
  • Explore the narrative structure of popular Manga
Warning‚ùå Bad examples of tasks
  • To understand the data
  • To learn about Chemistry

4. Identify one or more datasets

These can be exemplars; you are not locked in. Make sure the dataset is available, documented, and of good quality.

Tip‚úÖ Good examples of datasets
  • Bird Migration Data (data was used for a published paper, documentation exists, readable .csv format)
  • Cityscapes Dataset (benchmark dataset used for many CV/AI/ML papers, lots of code available on internet to process)
  • Iris Dataset (another canonical dataset, straight from an academic lab)
Warning‚ùå Bad examples of datasets
  • We will scrape Instagram (you probably won‚Äôt have time, data will be messy)
  • We will collect a survey about people‚Äôs habits (designing a good survey is super tricky, takes time)
  • We will request Twitter/X API developer access to collect Tweets (again, no time)
  • Anything like this: test.csv

5. Find, link, and briefly describe related work

These should be a few related visualizations you‚Äôve come across while looking into this area. You may want to consider how might a different kind of visualization reveal new insights?

Related work can be ‚Äúrelated‚Äù by being similar in the What-Why-How framework (Chapter 1,2,3) in any of these ways:

TipWays that related work can be related
  • Same/similar domain
  • Same/similar dataset type/semantics (Chapter 2)
  • Similar tasks & target user (Chapter 3)
  • Similar vis type/idiom (Chapter 5, Chapter 7, Chapter 8, Chapter 9)
  • Similar interaction mechanisms (Chapter 11-14)

You can use any visual you find on the internet or in print, but you must be able to trace it back to the original source and cite it.

Posting Your Pitch

Post your pitch (450 words maximum) in the #project-pitches Discord channel.

  1. Give your pitch an informative title
  2. Very briefly introduce yourself + any info you‚Äôd like to share about yourself
  3. Add a compelling (and related) image that will make people want to click on it to read more. This could be from the site/journal paper where you got the dataset, some of the related works you found, a sketch you drew, a relevant meme, etc.
  4. Summarize your idea that results from completing steps 1-5 above (domain, users, tasks, related work, data). Include links to examples, data, papers, etc.
  5. Select a relevant tag, or if none fit well, ask me to add a new one!
